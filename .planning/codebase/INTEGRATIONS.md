# External Integrations

**Analysis Date:** 2026-05-22

## APIs & External Services

**No third-party APIs detected.** This application is fully self-contained. All data is stored in Cloudflare D1 and served via internal Next.js API routes. No external HTTP calls are made to third-party services.

## Data Storage

**Databases:**
- Cloudflare D1 (SQLite-compatible serverless database)
  - Binding: `DB` (declared in `wrangler.toml`, typed in `src/types/cloudflare-env.d.ts`)
  - Access: via `getCloudflareContext().env.DB` in `src/lib/db.ts`
  - Query style: Raw SQL prepared statements (`db.prepare(...).bind(...).run()`)
  - Schema defined in `src/lib/schema.ts` using Drizzle ORM table definitions, but queries at runtime bypass the ORM and use raw SQL directly
  - Tables: `users`, `checkins`, `posts`, `post_likes`
  - Local dev DB: `./local.db` (referenced in `drizzle.config.ts` for migrations)

**File Storage:**
- None. No file uploads, no S3/blob storage, no Cloudflare R2 bindings.

**Caching:**
- None. OpenNext config sets `incrementalCache`, `tagCache`, and `queue` to `"dummy"` -- no caching layer.

## Authentication & Identity

**Auth Provider:**
- Custom, self-implemented JWT authentication
  - Implementation: `src/lib/auth.ts`
  - Password hashing: PBKDF2 with SHA-256, 100k iterations, random 16-byte salt (uses Web Crypto API `crypto.subtle`)
  - Token: Custom HS256 JWT signed with Web Crypto API (NOT using the `jose` library despite it being a dependency)
  - Token delivery: HTTP-only secure cookie named `token` (30-day expiry, `sameSite: lax`)
  - JWT payload: `{ sub: userId, iat, exp }` -- 30-day lifetime
  - Middleware guard: `src/middleware.ts` -- redirects unauthenticated users to `/login`, redirects authenticated users away from `/login` and `/register`
  - API route auth: Each protected API route calls `getUser(db, request, secret)` to extract and verify the JWT from cookies

**No OAuth, no social login, no third-party identity provider.**
